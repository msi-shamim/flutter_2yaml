Flutter .dart to compact YAML
for LLM/AI agents.

Convert Flutter widget files into token-efficient YAML with pipe syntax, CSS-like shorthands, and arrow callbacks — saving 50-70% tokens. Bidirectional: Dart ↔ YAML.

dart pub global activate flutter_2yaml

01. Why flutter_2yaml?

When feeding Flutter code to AI models as context, full .dart files waste tokens on boilerplate — import statements, @override, super.key, BuildContext context, const keywords, semicolons, curly braces, class declarations, and createState(). None of these carry meaningful UI information.

flutter_2yaml strips all boilerplate and produces structured YAML using an aggressive shorthand system — pipe syntax for Text/Image/Icon, CSS-like abbreviations for properties, and arrow notation for callbacks.

Before — Dart (45 lines)

class SplashScreen extends StatefulWidget {
  const SplashScreen({super.key});
  @override
  State<SplashScreen> createState()
    => _SplashScreenState();
}

class _SplashScreenState
    extends State<SplashScreen> {
  bool _isLoading = true;

  @override
  void initState() {
    super.initState();
    Future.delayed(
      const Duration(seconds: 3));
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: Colors.white,
      body: Center(
        child: Column(
          mainAxisAlignment:
            MainAxisAlignment.center,
          children: [
            Image.asset('assets/logo.png',
              width: 200, height: 200),
            const SizedBox(height: 20),
            if (_isLoading)
              const CircularProgressIndicator(
                color: Colors.blue),
          ],
        ),
      ),
    );
  }
}

After — YAML (14 lines)

page: SplashScreen
type: StatefulWidget
state: [_isLoading: bool = true]
lifecycle:
  initState: [Future.delayed(3s)]
build:
  Scaffold:
    bg: white
    body:
      Center:
        child:
          Column(center):
            children:
              - Image.asset: assets/logo.png
                  | 200x200
              - SizedBox: { h: 20 }
              - if _isLoading:
                  CircularProgressIndicator:
                    { color: blue }
50-70% token reduction The same widget structure in dramatically fewer tokens — every shorthand is designed to preserve full semantic understanding for LLMs.

02. Getting Started

Prerequisites

Dart SDK 3.8.0 or later. A Flutter project with .dart widget files.

Install

dart pub global activate flutter_2yaml

Convert a file

flutter_2yaml lib/screens/splash_screen.dart

Output: splash_screen.yaml in the same directory.

Convert a directory

flutter_2yaml lib/screens/ --recursive

Watch mode

flutter_2yaml lib/screens/ --watch

Auto-regenerates YAML whenever a .dart file is saved.

Non-widget files are skipped Services, repositories, models, and other non-widget classes are automatically ignored — only StatelessWidget and StatefulWidget (and framework variants) are converted.

04. YAML Format

Header

page: SplashScreen                    # 'page' if has Scaffold
widget: WelcomeCard                   # 'widget' for components
type: StatefulWidget                  # StatefulWidget or StatelessWidget
controller: HomeController(GetX)      # Auto-detected state management
state: [_isLoading: bool = true]      # Compact state variables

Lifecycle

lifecycle:
  initState: [Future.delayed(3s)]
  dispose: []

Named Children (Scaffold)

Scaffold:
  bg: white
  appBar:
    AppBar:
      title: Text: "Home" | 20 | bold
  drawer:
    Drawer: { ... }
  body:
    Center: { ... }
  floatingActionButton:
    FloatingActionButton:
      onPressed → add()
      child: Icon: add
  bottomNavigationBar:
    BottomNavigationBar: { ... }

Conditional & Collection Elements

children:
  - if _isLoading: CircularProgressIndicator: {}
  - ...items                              # Spread
  - for(item in products): Card: {}       # For-in

05. Pipe Syntax

Compress common widgets into a single line using | separators.

Text

Order: content | fontSize | fontWeight | color

YAMLDart Equivalent
Text: "Hello"const Text('Hello')
Text: "Hello" | 20Text('Hello', style: TextStyle(fontSize: 20))
Text: "Hello" | 20 | bold | whiteText('Hello', style: TextStyle(fontSize: 20, fontWeight: FontWeight.bold, color: Colors.white))
Text: title | 24Text(title, style: TextStyle(fontSize: 24))

Image

YAMLDart Equivalent
Image.asset: logo.png | 200x200Image.asset('logo.png', width: 200, height: 200)
Image.asset: banner.png | fit: coverImage.asset('banner.png', fit: BoxFit.cover)

Icon & SizedBox

YAMLDart Equivalent
Icon: menuconst Icon(Icons.menu)
SizedBox: { h: 20 }const SizedBox(height: 20)

08. Reverse Conversion

Convert compact YAML back to idiomatic Flutter Dart code.

flutter_2yaml reverse splash_screen.yaml
flutter_2yaml reverse yaml_output/ --recursive
flutter_2yaml reverse yaml_output/ --output lib/screens/

What Gets Generated

  • import 'package:flutter/material.dart'; always included
  • State management imports auto-added when controller: present
  • const constructors with super.key
  • @override annotations on lifecycle and build methods
  • super.initState() / super.dispose() in lifecycle
  • All shorthands expanded: bg → backgroundColor, p → EdgeInsets.all, etc.
Round-trip limitations Comments, complex callback bodies, and exact const placement may differ from the original. The structure and logic are preserved — formatting details may vary.

State Management

package:flutter_mobx/MobXstateManagement: MobX

Page vs Widget

Widgets containing Scaffold, CupertinoPageScaffold, MaterialApp, or CupertinoApp are classified as page:. All others use widget:.

Icon & Color

Icons.menu → menu, Colors.blue → blue, Color(0xFF123456) → #123456, double.infinity → full

10. Architecture

Forward:  .dart → DartFileAnalyzer → DartFileModel → YamlGenerator → .yaml
Reverse:  .yaml → YamlParser       → DartFileModel → DartGenerator → .dart
lib/src/
├── analyzer/                    # Forward: Dart AST → Model
│   ├── dart_file_analyzer.dart  # Main analyzer + state mgmt detection
│   ├── widget_visitor.dart      # RecursiveAstVisitor for widget tree
│   └── state_visitor.dart       # State variables, lifecycle methods
├── parser/                      # Reverse: YAML text → Model
│   ├── yaml_parser.dart         # Full YAML → DartFileModel
│   ├── widget_parser.dart       # Indentation-based tree parser
│   ├── pipe_parser.dart         # Text, Image, Icon pipe parsing
│   └── reverse_shorthand_map.dart
├── generator/                   # Output generation
│   ├── yaml_generator.dart      # Model → compact YAML
│   ├── dart_generator.dart      # Model → idiomatic Dart
│   └── shorthand_map.dart       # Forward shorthand lookups
├── models/                      # Shared data structures
│   ├── dart_file_model.dart     # Complete file representation
│   ├── widget_node.dart         # Widget tree node
│   └── state_model.dart         # State, lifecycle, methods
├── cli/
│   ├── command_runner.dart      # CLI argument parsing
│   └── watcher.dart             # File watch mode
├── converter.dart               # Forward file I/O
└── reverse_converter.dart       # Reverse file I/O

12. Example Combinations

Quick scan of all screens

flutter_2yaml lib/screens/ --recursive --level minimal

Full context for AI agent

flutter_2yaml lib/ --recursive --level standard --output .ai/context/

Watch during development

flutter_2yaml lib/screens/ --watch --level standard

Round-trip: edit YAML, regenerate Dart

# Generate YAML
flutter_2yaml lib/screens/home_screen.dart --output yaml/

# Edit yaml/home_screen.yaml with AI or manually...

# Regenerate Dart
flutter_2yaml reverse yaml/home_screen.yaml --output lib/screens/

13. Programmatic API

import 'package:flutter_2yaml/flutter_2yaml.dart';

// Forward: Dart → YAML
final analyzer = DartFileAnalyzer();
final model = analyzer.analyze(dartSource, 'my_widget.dart');
final yamlGenerator = YamlGenerator();
final yaml = yamlGenerator.generate(model!, VerbosityLevel.standard);

// Reverse: YAML → Dart
final yamlParser = YamlParser();
final reverseModel = yamlParser.parse(yamlSource, 'my_widget.yaml');
final dartGenerator = DartGenerator();
final dartCode = dartGenerator.generate(reverseModel);

// File-level conversion
final converter = Converter();
converter.convertFile('lib/screens/home.dart');
converter.convertDirectory('lib/screens/', recursive: true);

final reverseConverter = ReverseConverter();
reverseConverter.reverseFile('home.yaml');
